Why this matters

RL patterns work after training too

Reinforcement learning from human feedback (RLHF) is how frontier models improve: humans rank outputs, those preferences update model weights, the model gets better. The core pattern is simple — correction signal in, improved behavior out.

That same pattern works at inference time, without touching weights. When an operator's correction is prepended as context, the agent avoids the same mistake — not because it was retrained, but because it was informed. Our eval results show this empirically: +1.7 average improvement in aquaculture tasks where the model had genuine knowledge gaps.

This means the correction signal that frontier labs capture internally through millions of user conversations can also be captured externally, by operators, and applied at the context layer. The correction is the same data; the application mechanism is different.

Helping open systems compete

Frontier labs have a closed loop that open-source can't replicate:

User interactions → corrections → RLHF → better model → more users → more corrections

This flywheel is proprietary. Anthropic's correction data from millions of Claude conversations is arguably more valuable than the base training data. Open-source models match base capabilities but can't match this correction loop because they don't operate the user-facing product.

Synthesis creates a public correction layer. Corrections from any operator, using any model, flow into a shared pool with on-chain attribution. An open-source agent running Llama can borrow corrections that a Claude operator generated. The protocol is model-agnostic and lab-agnostic. No single entity controls the correction commons.

This doesn't replace RLHF — some improvements fundamentally require weight updates. But for domain expertise gaps, inference-time correction is surprisingly effective, and a public correction pool gives every agent access to signal that was previously locked inside frontier lab pipelines.

The ratio economy: lessons from private trackers

The credit system draws from what actually worked in peer-to-peer file sharing networks like What.CD and Karagarga — communities that solved the free-rider problem without financialization.

In those systems, your ratio (upload / download) determined your standing. Good ratio meant more access, higher trust, invitation privileges. Bad ratio meant restrictions. Nobody paid money — they contributed bandwidth. The system was self-regulating: generous participants got more out of it, free-riders gradually lost access.

Synthesis applies the same model to agent corrections:

  • Contribute corrections → your balance goes up → larger credit line → you can borrow more
  • Only borrow → balance goes negative → credit line restricts → you need to contribute back
  • Reputation from measured deltas → high-quality contributors get extended credit, like trusted uploaders getting ratio bonuses

The key insight from those communities: the ratio isn't a price, it's a social contract. It says "I participate in this commons." Credits in Synthesis work the same way — they're not a currency to accumulate, they're a measure of reciprocity.

Liberating memory

Every agent system generates memory. Claude Code writes feedback files. Codex captures steering events. Cursor has shadow workspaces. Windsurf has flows. Karpathy's autoresearch showed that program.md — the accumulated human corrections — is the most valuable artifact in an autonomous research loop.

All of this expertise is local. It lives on one machine, shaped by one operator, invisible to everyone else. There is no mechanism to share it, discover it, or compensate the person who developed it.

We surveyed every major open-source agent memory project (A-Mem, MemOS, Acontext, Mem0, Letta). None of them publish actual memory content. Memory is treated as ephemeral and private — generated at runtime, never persisted as a shareable artifact. The infrastructure exists; the sharing doesn't.

The incentive problem is real: why would an operator share corrections that cost them time and expertise to produce? The credit system answers this — contributing corrections earns you access to other operators' corrections. The more you give, the more you can take. Combined with on-chain attribution (your ERC-8004 identity is permanently linked to the corrections you contributed), there's both a practical incentive (credit) and a reputational one (provenance).
